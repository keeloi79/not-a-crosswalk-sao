🤝 Contributing to Not Just a Crosswalk We welcome contributions that expand the visibility and impact of this project. Whether you are adding a new community flag or a high-energy animation, follow these guidelines to ensure your code remains efficient and "conference-hardened".  🏳️‍🌈 Adding a New Flag (Palette) Flags are static color patterns stored in the LED buffer. To add a new one, follow these steps:  1. Define the Flag Function In LED_functions.h, create a new function that uses LED_SendColour to fill the 18-LED array. Use the global BRIGHT variable to ensure the user's brightness preference is respected.  C void LED_YOUR_FLAG_NAME(void) {     // Example: 6 LEDs per color for a 3-stripe flag     for (int i = 0; i < 6; i++) LED_SendColour(R, G, B, BRIGHT);     for (int i = 0; i < 6; i++) LED_SendColour(R, G, B, BRIGHT);     for (int i = 0; i < 6; i++) LED_SendColour(R, G, B, BRIGHT);     Delay_Us(50); // Latch signal } 2. Register in the Master Renderer Add your new function to the switch statement inside SHOW_CURRENT_FLAG. Assign it the next available case number.  3. Update the Limit In ch32v00x_it.c, increment the MAX_PALETTES define to include your new flag. This ensures the button interrupt can cycle to your new creation.  ⚡ Adding a New Animation Effect Animations are active loops that manipulate the LED buffer in real-time.  1. Create the Effect Function Define your effect in LED_functions.h. Your function must include a check for effect_index != current_mode to ensure it stops running as soon as a button is pressed.  No Floats: Use integer math only to save Flash space.  Non-Blocking: Use Delay_Ms() for timing so the system remains responsive.  Power Awareness: Be mindful of current draw; use the hardware-stable floor (minimum brightness of 3) to prevent flickering.  2. Register in the Effect Engine Add your effect to the switch statement in RUN_SELECTED_EFFECT.  3. Update the Limit Increment MAX_EFFECTS in ch32v00x_it.c.  📏 Technical Style Guide Persistence: Do not modify the SAVE_SETTINGS or LOAD_SETTINGS logic unless necessary, as these are critical for the mission of digital resistance.  Flash Economy: Always check your build size. We aim to keep the firmware under 12KB to leave room for future community flags.  Timing: Standardize your animations at 25ms or 50ms intervals to match the badge's established "feel" and electrical profile.  📬 Submission Process Fork the repository.  Test your changes on physical hardware (Not Just a Crosswalk SAO v8 or compatible CH32V003 setup).  Submit a Pull Request with a description of the flag or effect and why it should be added to the project.  This project is as much about the community as it is about the code. Thank you for helping keep the colors visible.
